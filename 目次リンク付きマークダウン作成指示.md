# 目次リンク付きマークダウン作成指示

> **使い方:** 目次リンク付きの Markdown を作りたい「原文」と、この指示文書を AI（Claude / Cursor など）に読み込ませてください。AI はこの指示に従い、クリックでジャンプする目次付きの Markdown を出力します。
> **Cursor 利用者向け:** この文書の「Cursor での一度だけの設定」も実施すると、プレビュー画面で目次リンクが動作します。

---

## 目次

> **プレビューで目次リンクをクリックしてジャンプするには:** 拡張機能 **Markdown Preview Enhanced** をインストールし、そのプレビューで開いてください（ファイルを右クリック → **Open Preview** または **Ctrl+Alt+M**）。標準の Cursor プレビューでは目次リンクは動作しません。

## 1. この指示の目的

- 作成する Markdown に**目次（Table of Contents）**を付け、**目次の各項目をクリックすると該当セクションにジャンプする**ようにする。
- Cursor のプレビューでは標準機能では目次リンクが動かないため、**拡張機能のインストール**と**ショートカットの割り当て**までこの文書に含める。二度と調べなくてよいようにする。

---

<a id="doc-2"></a>
## 2. Markdown の書き方（AI が従うルール）

<a id="doc-2-1"></a>
### 2.1 目次の位置と形式

- 文書の冒頭（タイトルやリード文の直後）に **「## 目次」** を置く。
- 目次は **リンク付きリスト** とする。形式は次のとおり。

```markdown
## 目次

- [セクション名](#アンカーID)
- [別のセクション](#別のアンカーID)
    - [サブセクション](#サブアンカーID)
```

- アンカーID は **半角英数字とハイフンのみ** を使う（例: `sec0`, `sec1-1`, `ref2-3`）。`#` は書かない。

<a id="doc-2-2"></a>
### 2.2 ジャンプ先の付け方（必須）

- **目次でリンクするすべての見出しの直前に**、同じアンカーID の **HTML アンカー** を 1 行で書く。
- 見出しには **`{#id}` のような Pandoc 記法は使わない**。Cursor 標準プレビューでは解釈されないため。

**正しい例:**

```markdown
<a id="sec0"></a>
## 0. セットアップ

<a id="sec1-1"></a>
### 1.1 サブセクションのタイトル
```

- 目次に `[0. セットアップ](#sec0)` と書いたなら、本文には必ず `<a id="sec0"></a>` を直後の見出しの前に置く。

<a id="doc-2-3"></a>
### 2.3 目次直後の「プレビューで見る人向け」注記

目次のすぐ下に、次の内容を **引用ブロック（`>`）** で 1〜2 行で入れる。

```markdown
> **プレビューで目次リンクをクリックしてジャンプするには:** 拡張機能 **Markdown Preview Enhanced** をインストールし、そのプレビューで開いてください（ファイルを右クリック → **Open Preview** または **Ctrl+Alt+M**）。目次リンクが同じ文書内でジャンプします。標準の Cursor プレビューでは目次リンクは動作しません。
> **拡張を使わない場合:** 「Markdown」タブをアクティブにして左の「アウトライン」を使うか、**Ctrl+F** で検索してください。
```

- ショートカットを **Ctrl+Alt+M** にしている前提で書いてある。ユーザーが別のキーにしている場合は読み替える。

<a id="doc-2-4"></a>
### 2.4 禁止事項

- 目次リンクに **フラグメントのみ**（例: `[見出し](#sec0)`）だけを書いて、本文に `<a id="sec0"></a>` を付けない → **禁止**。必ずセクション直前にアンカーを置く。
- 見出し行に **`## 0. セットアップ {#sec0}`** のように `{#id}` を付けるだけにしない。Cursor 標準プレビューでは無視されるため、**`<a id="sec0"></a>` を直前の行に書く**。

## 3. Cursor での一度だけの設定（利用者が手動で行う）

目次リンクを **プレビュー画面でクリックしてジャンプ** できるようにするための設定。**初回のみ** 行えばよい。

<a id="doc-3-1"></a>
### 3.1 拡張機能の追加

1. **Ctrl+Shift+X** で拡張機能ビューを開く。
2. 検索欄に **「Markdown Preview Enhanced」** と入力。
3. 公開者 **shd101wyy** の **Markdown Preview Enhanced** を選び、**インストール** する。

<a id="doc-3-2"></a>
### 3.2 ショートカットの割り振り直し（推奨）

- 標準の「プレビューを横に開く」は **Ctrl+K V** などだが、**Ctrl+Shift+V** は「貼り付け」でよく使うため、バッティングすることがある。
- **Markdown Preview Enhanced: Open Preview to the Side** に、**Ctrl+Alt+M** を割り当てることを推奨する（M = Markdown の頭文字）。

**手順（キーボードショートカット画面で編集する方法）:**

1. **Ctrl+Shift+P** でコマンドパレットを開き、**「キーボード ショートカットを開く」** または **「Open Keyboard Shortcuts」** を実行する。
2. 検索欄に **「Markdown Preview Enhanced」** と入力し、一覧から **「Markdown: Markdown Preview Enhanced: Open Preview to the Side」** の行を表示する。
3. その行の **キーバインド（例: Ctrl K V）の左側にある「えんぴつ（編集）」アイコン** をクリックする。  
   ※ **行全体をクリックするとプレビューが開いてしまうので、キー割り当ての左のえんぴつアイコンだけを押す。**
4. 使いたいキーを押す（例: **Ctrl+Alt+M**）。
5. **Enter** で確定する。

これで、Markdown ファイルを開いた状態で **Ctrl+Alt+M** を押すと、Markdown Preview Enhanced のプレビューが横に開く。そのプレビューでは目次リンクのクリックで同じ文書内にジャンプする。

**別の方法（keybindings.json で追加する）:**

1. **Ctrl+Shift+P** で **「Open Keyboard Shortcuts (JSON)」** を実行。
2. 開いた `keybindings.json` の配列 `[ ]` の中に、次の 1 ブロックを追加して保存する。

```json
{
  "key": "ctrl+alt+m",
  "command": "markdown-preview-enhanced.openPreviewToTheSide",
  "when": "editorLangId == 'markdown'"
}
```

<a id="doc-3-3"></a>
### 3.3 プレビューの開き方（設定後）

- **Ctrl+Alt+M** … 横に Markdown Preview Enhanced のプレビューを開く（推奨）。
- または ファイル名を **右クリック → Open Preview**。

<a id="doc-4"></a>
## 4. まとめチェックリスト（AI 用）

Markdown を出力するとき、以下を満たしているか確認する。

- [ ] 目次を `## 目次` の下に、`[表示テキスト](#アンカーID)` の形式で書いた。
- [ ] 目次で参照するすべてのセクションの**直前に** `<a id="アンカーID"></a>` を 1 行ずつ置いた。
- [ ] 目次の直下に、プレビューで目次リンクを使う方法（拡張機能と Ctrl+Alt+M）の注記を引用ブロックで入れた。
- [ ] 見出しに `{#id}` だけを付け、`<a id="..."></a>` を省略していない。
